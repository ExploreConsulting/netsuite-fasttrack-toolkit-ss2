Class AddressBase

The addressbook 'subrecord'. In SS2.0 this is mostly treated as a normal record object. However I caution against trying to create new instances of it, only passing an existing record to the constructor. For example, on the customer record you can get an address sublist record with Record.getSublistSubrecord() then pass the returned record to the Address constructor. Currently this has only been tested for read access to aqddress properties defined below.
